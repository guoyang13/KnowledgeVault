# DDD学习任务清单

> 这份清单用于把《实现领域驱动设计》读成自己的能力。每完成一项，就在 [[01-初学者阅读日志]] 里记录一次。

## 每章检查题

### [[01-第01章-DDD入门]]

- DDD 解决的核心问题是什么？
- 为什么只用 Entity/Repository 不是完整 DDD？
- 业务专家在 DDD 中有什么作用？

### [[02-第02章-领域子域和限界上下文]]

- 核心域和支撑子域怎么区分？
- 限界上下文和微服务是什么关系？
- 为什么同一个词在不同上下文中可以有不同含义？

### [[03-第03章-上下文映射图]]

- 什么情况下需要防腐层？
- 发布语言和开放主机服务有什么区别？
- 为什么上下文映射不是部署架构图？

### [[04-第04章-架构]]

- 六边形架构解决什么问题？
- 应用服务和领域服务如何分工？
- CQRS 适合什么场景，不适合什么场景？

### [[05-第05章-实体]]

- 实体的身份是什么？
- 为什么实体不应该只有 getter/setter？
- 数据库主键和业务身份有什么区别？

### [[06-第06章-值对象]]

- 值对象为什么建议不可变？
- Money、Address、DateRange 为什么是值对象？
- 值对象如何减少实体滥用？

### [[07-第07章-领域服务]]

- 什么样的行为适合放进领域服务？
- 领域服务和应用服务有什么区别？
- 为什么领域服务不应该变成万能 service？

### [[08-第08章-领域事件]]

- 命令和事件有什么区别？
- 事件应该包含哪些信息？
- 领域事件如何支持最终一致性？

### [[09-第09章-模块]]

- 为什么按业务概念组织包更好？
- 技术分包有什么问题？
- 模块边界如何体现通用语言？

### [[10-第10章-聚合]]

- 聚合保护什么？
- 为什么聚合要尽量小？
- 为什么聚合之间优先用 ID 引用？

### [[11-第11章-工厂]]

- 什么时候需要工厂？
- 工厂和构造函数有什么区别？
- 工厂如何保护创建规则？

### [[12-第12章-资源库]]

- Repository 和 DAO 的区别是什么？
- 为什么 Repository 应面向聚合？
- 复杂查询应该放在哪里？

### [[13-第13章-集成限界上下文]]

- 哪些集成适合同步 API？
- 哪些集成适合领域事件？
- 防腐层应该放在哪一侧？

### [[14-第14章-应用程序]]

- 应用服务负责什么？
- DTO 和领域对象怎么隔离？
- 业务规则为什么不应该主要写在应用服务里？

## 项目实战交付物

最终你应该产出这些东西：

| 交付物 | 标准 |
|---|---|
| 子域划分表 | 至少包含核心域、支撑子域、通用子域 |
| 限界上下文列表 | 每个上下文有自己的核心语言 |
| 上下文映射图 | 每条关系有关系类型和集成方式 |
| 聚合设计说明 | 包含聚合根、不变量、命令、事件 |
| Java 包结构 | 按上下文和业务概念组织 |
| 单元测试 | 覆盖关键业务规则和领域事件 |
| 防腐层设计 | 外部模型不直接进入领域模型 |

## 学习误区自检

- 我是不是把 DDD 当成目录结构？
- 我是不是还在按表设计实体？
- 我是不是把所有逻辑写进 service？
- 我是不是没有识别核心域？
- 我是不是没有画上下文映射？
- 我是不是让外部系统 DTO 污染了本地模型？
- 我是不是把查询方便当成聚合边界？
- 我是不是试图用一个大事务解决所有一致性？
