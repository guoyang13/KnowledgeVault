# 5.3.1. ABSD概念要点

> 本节要点来自课件「基于架构的软件开发方法 - 概念」。**ABSD**（Architecture-Based Software Development）即基于架构的软件开发方法。上级：[[5.3. 基于架构的软件开发方法]]。

### 概念要点

1. **架构驱动**：ABSD 方法强调由**业务（商业）**、**质量**和**功能需求**的组合共同**驱动架构设计**。
2. **三个基础**：
   - **功能的分解**：在分解中沿用基于**模块**的**内聚**与**耦合**技术；
   - **通过选择架构风格**来满足**质量**与**业务**需求；
   - **软件模板**的运用。
3. **视角与视图**：从**不同的视角**检查系统，因而得到**不同的视图**（与 4+1 等视图思想一致，可对照 [[5.2.4. 软件架构4+1视图]]）。
4. **需求捕获方式**：**用例**用于**捕获功能需求**；特定场景以 **刺激、环境、响应** 三要素描述，用于**捕获质量需求**。
