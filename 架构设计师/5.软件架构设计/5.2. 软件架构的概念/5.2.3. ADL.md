# 5.2.3. ADL

> 本节要点来自课件「架构描述语言 ADL」。**ADL**（Architecture Description Language）即架构描述语言。上级：[[5.2. 软件架构的概念]]。

### 定义

**ADL** 是这样一种**形式化语言**：在**底层语义模型**的支持下，为软件系统的**概念体系结构**建模提供**具体语法**和**概念框架**。

### 常见 ADL 举例

| 名称 | 特点（课件表述） |
| --- | --- |
| **C2SADL** | 基于**组件**和**消息**的软件架构描述语言 |
| **Wright** | **分布、并发**类型的架构描述语言 |
| **ACME** | **架构互换**语言 |
| **UniCon** | 基于**组件**和**连接**的架构描述语言 |
| **Rapide** | 基于**事件**的架构描述语言 |
| **其他** | Darwin、MetaH、Aesop、Weaves、SADL、xADL |

### UniCon 示例（课件代码片段）

```text
USES p1 PROTOCOL Unix-pipe
USES sorter INTERFACE Sort-filter
CONNECT sorter.output TO p1.source
USES p2 PROTOCOL Unix-pipe
USES printer INTERFACE Print-filter
CONNECT sorter.input TO p2.sink
```

### ADL 的三个基本元素

1. **构件**：计算或数据存储单元。
2. **连接件**：用于构件之间交互建模的体系结构构造块，以及**支配这些交互的规则**。
3. **架构配置**：描述体系结构的**构件与连接件的连接图**。
