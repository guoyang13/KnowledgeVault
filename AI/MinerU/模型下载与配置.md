# MinerU：模型下载与模型源配置

> 摘要自官方文档，便于本地查阅。权威说明以官方为准：  
> - [Model Source](https://opendatalab.github.io/MinerU/usage/model_source/)  
> - [CLI Tools](https://opendatalab.github.io/MinerU/usage/cli_tools/)  
> - [Quick Start](https://opendatalab.github.io/MinerU/quick_start/)

## 1. 模型存在哪里、由谁提供

MinerU 从远程仓库拉取模型，支持：

| 源               | 说明                                                 |
| --------------- | -------------------------------------------------- |
| **HuggingFace** | **默认**。全球访问速度与稳定性较好。                               |
| **ModelScope**  | **中国大陆用户推荐**。与 `hf` 生态兼容，适合无法稳定访问 HuggingFace 的场景。 |

也可将模型下载到本机后，通过 **`local`** 模式使用（见下文）。

---

## 2. 切换模型源：环境变量 `MINERU_MODEL_SOURCE`

MinerU **不再提供**单独的「命令行开关」来选模型源，统一用环境变量；对 **所有命令行工具与 API 调用** 生效。

### 2.1 临时（当前终端会话）

**使用 ModelScope（国内常用）：**

```bash
export MINERU_MODEL_SOURCE=modelscope
```

**使用 HuggingFace（默认，可不设）：**

```bash
export MINERU_MODEL_SOURCE=huggingface
# 或不设置该变量（即默认 HuggingFace）
```

之后在**同一个终端**里运行 `mineru`、`mineru-models-download` 等即可。关闭终端或修改变量后失效。

### 2.2 在代码里设置（示例）

```python
import os
os.environ["MINERU_MODEL_SOURCE"] = "modelscope"
```

### 2.3 与解析命令一起用（示例）

```bash
export MINERU_MODEL_SOURCE=modelscope
mineru -p <输入路径> -o <输出路径>
```

## 3. 下载模型到本地：`mineru-models-download`

### 3.1 先看帮助

```bash
conda activate mineru   # 若使用独立 conda 环境
mineru-models-download --help
```

### 3.2 交互式下载（推荐第一次使用）

```bash
export MINERU_MODEL_SOURCE=modelscope   # 国内建议先设；若用 HF 可改为 huggingface 或不设
mineru-models-download
```

按终端提示选择要下载的模型组件。下载完成后：

- 终端会**输出模型路径**；
- 配置会**自动写入用户目录下的 `mineru.json`**。

### 3.3 官方说明要点（`mineru.json` 与更新）

- 也可将仓库中的 [配置模板 `mineru.template.json`](https://github.com/opendatalab/MinerU/blob/master/mineru.template.json) 复制到用户目录并改名为 **`mineru.json`**，再按需改路径。
- 模型下载到本地后，可将模型文件夹**移动到其他位置**，并同步修改 **`mineru.json`** 中的模型路径。
- 若把模型部署到另一台机器，需把 **`mineru.json`** 放到新机器对应用户目录，并保证路径正确。
- **更新模型**：可再次运行 `mineru-models-download`。  
  - 若**未移动**本地模型目录：一般为**增量更新**。  
  - 若**已移动**模型目录：可能**重新下载到默认位置**并更新 `mineru.json`（以当前版本行为为准）。
- **`mineru-models-download` 必须从远程源真实下载**。若当前 shell 已设置 `MINERU_MODEL_SOURCE=local`，该命令会**暂时忽略**该值，仍按你选择的 **huggingface / modelscope** 执行下载。

---

## 4. 使用本机已下载的模型

解析时改为使用本地模型：

```bash
export MINERU_MODEL_SOURCE=local
mineru -p <输入路径> -o <输出路径>
```

前提是 **`mineru.json` 中模型路径** 已指向本机有效目录（通常由 `mineru-models-download` 写好）。

---

## 5. 自定义配置文件路径（可选）

若不想用默认用户目录下的 `mineru.json`，可通过环境变量指定：

```bash
export MINERU_TOOLS_CONFIG_JSON="/path/to/your/mineru.json"
```

（说明见官方 [CLI Tools - 环境变量](https://opendatalab.github.io/MinerU/usage/cli_tools/) 中 `MINERU_TOOLS_CONFIG_JSON`。）

---

## 6. 建议操作顺序（conda 环境 `mineru`）

1. `conda activate mineru`
2. （国内）`export MINERU_MODEL_SOURCE=modelscope`
3. `mineru-models-download` 按提示完成下载，确认终端输出路径、检查用户目录 **`mineru.json`**
4. 解析：  
   - 继续用远程拉取：`export MINERU_MODEL_SOURCE=modelscope`（或 huggingface）后执行 `mineru -p ... -o ...`  
   - 或仅用本地：`export MINERU_MODEL_SOURCE=local` 后执行 `mineru -p ... -o ...`

---

## 7. 排错提示

- **下载失败 / 超时**：换源（`modelscope` ↔ `huggingface`）、检查代理与磁盘空间。
- **`mineru` 找不到**：确认已 `conda activate mineru`（或对应 venv）。
- **版本差异**：不同 MinerU 版本子命令与选项可能变化，以 **`mineru-models-download --help`** 与官方文档为准。
