# 推理加速三件套：FlashAttention / PagedAttention / Speculative Decoding

> 当前 LLM 推理性能优化的三大支柱，几乎所有 serving 引擎都内置。

## 1. FlashAttention
- **解决**：标准 Attention 的内存 IO 瓶颈（HBM ↔ SRAM 来回搬数据）。
- **思路**：tiling + recomputation，把 Attention 计算保持在片上 SRAM 内。
- **效果**：训练/推理速度数倍提升，长序列尤其明显。
- **版本**：FlashAttention-1/2/3，FA-3 针对 Hopper（H100）做了 FP8 + 异步流水线。

## 2. PagedAttention（vLLM）
- **解决**：KV Cache 显存碎片化、利用率低。
- **思路**：借鉴 OS 分页管理，把 KV 切成固定大小 block，按需分配。
- **额外好处**：天然支持 **Prefix Sharing**（多请求共享相同前缀）。

## 3. Speculative Decoding（投机解码）
- **思路**：小模型快速猜 K 个 token，大模型一次性验证。
- **变体**：
  - **Medusa**：在主模型上加多个 head 同时预测后续 token。
  - **EAGLE**、**Lookahead Decoding**：更精细的猜测策略。
  - **Self-Speculative**：用同一模型不同层做 drafter。
- **效果**：2–3 倍解码速度，几乎无质量损失。

## 其他相关技术
- **Continuous Batching**：vLLM 的另一大优化，动态拼 batch。
- **Chunked Prefill**：长 prompt 分块处理，平衡 TTFT 与吞吐。
- **FP8 / INT8 Inference**：H100/Blackwell 上的关键加速。
