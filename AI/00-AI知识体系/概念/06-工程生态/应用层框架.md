# 应用层框架（LangChain / LangGraph / DSPy / 等）

> 把 LLM、工具、检索、Agent 组合起来构建应用的开发框架。选型取决于"你要构建的是 Workflow 还是 Agent"。

## 通用 LLM 应用框架
- **LangChain**：最早、生态最广，但被诟病"封装过厚"。新版更克制。
- **LlamaIndex**：偏 RAG / 文档处理，索引和召回工具丰富。
- **Haystack**（Deepset）：偏企业级搜索 + RAG。

## Agent 编排框架
- **LangGraph**：LangChain 的现代答案，**有状态图 + 持久化 + Human-in-the-loop**，事实标准之一。
- **OpenAI Agents SDK / Swarm**：轻量 Handoff 模型，适合多 Agent 协作。
- **CrewAI**：角色 + 任务定义友好，团队协作风格。
- **AutoGen**（Microsoft）：研究取向，多 Agent 对话与对抗。
- **Pydantic AI**：类型安全、结构化输出第一。

## 提示/程序优化
- **DSPy**（Stanford）：把 prompt 当成可学习的程序，自动优化 prompt 与 few-shot。理念领先但学习曲线陡。
- **Instructor**：基于 Pydantic 的结构化输出库。
- **Outlines**、**LMQL**：受约束生成 / DSL。

## 评测与可观测
- **LangSmith / LangFuse / Arize Phoenix / Helicone**：trace、eval、监控。
- **Inspect**、**Promptfoo**：评测取向。

## 选型建议
- **Workflow 类应用**：LangChain / LlamaIndex / Pydantic AI 任选。
- **复杂 Agent / 长任务**：LangGraph、OpenAI Agents SDK。
- **多角色协作**：CrewAI、AutoGen。
- **要追求最优 prompt**：DSPy。
- **不知道选啥**：很多产品最后是自己手写一个"薄薄的 harness"，少依赖框架。
