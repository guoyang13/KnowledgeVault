# 什么是 LLM

> **Large Language Model（大语言模型）**：在海量文本上预训练、参数量达数十亿以上的**自回归语言模型**；通过 API 或本地部署，以「续写文本」的方式完成对话、推理、代码、工具调用等任务。

## 核心定义

| 维度 | 说明 |
|------|------|
| **Large** | 参数量与训练数据规模大（常见 7B～数百 B）；算力与数据是能力上限的重要来源 |
| **Language** | 建模对象是**自然语言（及代码等文本）的 token 序列** |
| **Model** | 可学习的神经网络；输入前文，输出下一 token 的概率分布 |

## LLM 在做什么（一句话）

给定 token 序列 \(x_1,\ldots,x_t\)，学习 \(P(x_{t+1} \mid x_1,\ldots,x_t)\)，推理时**逐个采样**生成后续 token，直到结束符或长度上限。

## 与传统 NLP / 小模型的区别

| | 传统 NLP（规则 / 小模型） | LLM |
|--|---------------------------|-----|
| 任务形式 | 常为「一个任务一个模型」（分类、NER…） | **通用**：同一模型经提示或微调适配多任务 |
| 知识来源 | 特征工程 + 标注数据 | **预训练**从语料吸收语言与常识；RAG 可补实时知识 |
| 交互方式 | 固定输入输出 schema | **自然语言指令** + 可选工具 / JSON 模式 |
| 能力涌现 | 较弱 | 规模上去后出现 ICL、推理、工具使用等（非保证） |

## 能力边界（应用工程师必知）

- **擅长**：语言理解/生成、格式遵循、模式模仿、在上下文内的推理与综合
- **不擅长 / 需外挂**：实时事实、私有数据、精确计算、可验证执行 → 用 **RAG、工具、代码解释器**
- **幻觉**：高置信的错误陈述；不能单靠「更大模型」消除，需评测与约束
- **上下文窗口**：能「看见」的 token 有上限；超长文档要分块、摘要或检索
