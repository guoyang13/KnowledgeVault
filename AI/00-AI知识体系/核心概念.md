## Harness 一词多义辨析

`harness` 在 LLM 圈被借用了多次，看到时先判断在说哪一种：

| 类型 | 含义 | 笔记 / 实例 |
|---|---|---|
| **Eval Harness** | 跑评测集的统一框架 | [[Eval Harness]] |
| **Agent Harness / Scaffold** | 包装 LLM 成为 Agent 的代码层 | [[Agent Harness]] |
| **Benchmark Harness** | 特定基准自带的运行环境 | SWE-bench、WebArena、OSWorld |
| **Training Harness** | 训练 / 微调框架 | Megatron-LM、TRL、Axolotl（见 [[高效微调]]） |
| **Inference / Serving Harness** | 推理服务化脚手架 | vLLM、SGLang（见 [[推理加速三件套]]） |
| **Test Harness**（传统软工） | 单元/集成测试脚手架 | pytest、JUnit |

> 经验法则：**评测**语境多指 Eval Harness；**Agent / SWE-bench** 语境多指 Agent Harness；**训练论文**里多指 Training Harness。

## 维护与编号约定

### 编号规则
- **目录编号**：两位数字 + 短横线（`01-`、`02-`…），按主题分类。预留：
  - `08-` ~ `09-` 给未来新主题（如 `08-多模态进阶/`、`09-AI硬件/`）。
  - `99-Archive/` 给归档/废弃笔记。
- **文件编号**：每个概念笔记也用 `NN-` 前缀，顺序与本 MOC 的 `X.Y` 章节号一致。
- **双链使用别名**：每个文件 frontmatter 都声明 `aliases: [原名]`，因此 wikilink 仍写简洁的 `[[Transformer架构]]`，由 Obsidian 自动解析到 `01-Transformer架构.md`。**绝不要**写成 `[[01-Transformer架构]]`。

### 新增笔记
1. 在对应 `0X-...` 目录下，按当前编号续号创建 `NN-新概念.md`。
2. 文件顶部加 frontmatter：
   ```yaml
   ---
   aliases:
     - 新概念
   ---
   ```
3. 在本 MOC 对应章节末尾追加一行 `X.Y [[新概念]]`。

### 调整阅读顺序
- 想让某篇排前面 → 改文件名前缀编号（如 `05-x` → `02-x`），其他文件相应递增。
- 由于双链走 alias，**改文件名前缀不会破坏任何引用**。
- 也可以在文件名前缀间留空位（如 `01-`、`05-`、`10-`），后续插入新概念时无需重排。

### 跨主题概念
- 放在最契合的目录；在其他章节用 `[[xxx]]` 双链交叉引用即可。

### 归档 / 重命名 / 删除
- **归档**：移到 `99-Archive/`，保留文件名以维持历史双链。
- **重命名**：改文件名后，在 frontmatter `aliases` 里同时保留新旧两个名字，避免旧双链失效。
- **删除**：先用 Obsidian 反链面板确认没有引用，再删除。
