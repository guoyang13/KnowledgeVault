# 常见误区与学习检查

## 常见误区

1. **把本体当数据库表设计。** 本体表达领域语义，不只是存储结构。
2. **类和实例混淆。** `Supplier` 是类，`供应商A` 是实例。
3. **过早追求大而全。** 先围绕能力问题做最小可用本体。
4. **把所有规则都放进 OWL。** 数据质量用 SHACL，复杂业务流程用规则引擎或代码。
5. **忽视开放世界假设。** OWL 不知道某事实，不代表该事实为假。
6. **URI 不稳定。** URI 一旦被系统依赖，修改成本很高。
7. **让 LLM 自由解释术语。** Agent 应先查本体，再生成查询和解释。

## 学习目标检查

学完这一阶段，应能回答：

1. 本体和知识图谱有什么区别？
2. 本体、ER 模型、数据字典分别解决什么问题？
3. Class、Individual、ObjectProperty、DataProperty、Constraint 分别是什么？
4. RDF 三元组如何表达事实？
5. OWL 相比 RDF/RDFS 多解决了什么？
6. 为什么 OWL 的开放世界假设会影响建模？
7. SPARQL 如何查询图数据？
8. SHACL 为什么对企业数据质量重要？
9. 为什么 LLM Agent 需要本体来统一业务语义？
10. 在供应链场景里，如何定义“缺货风险”并让 Agent 按定义执行？

## 自测任务

试着不用看笔记，回答这三个问题：

```text
1. 如果库存低于安全库存，应该用 OWL、SHACL、规则引擎还是查询逻辑表达？
2. 如果一个订单没有承运商，为什么 OWL 不一定认为它违规？
3. 如果用户问“哪些 SKU 有缺货风险”，Agent 应该先查本体还是直接写 SQL？
```

如果这三个问题能讲清楚，说明你已经抓住本体建模的核心边界。
