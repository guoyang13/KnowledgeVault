# 阶段 0：问题意识

## 本体解决什么问题

本体不是为了“更高级地画图”，而是为了减少这些问题：

- 不同系统对同一个词口径不同。
- LLM/Agent 对业务词想当然理解。
- 数据表字段能查，但业务含义不清楚。
- 指标计算散落在 SQL、Excel、代码和口头约定里。
- 新系统接入时，需要反复解释业务概念。

## 典型业务痛点

以供应链为例，不同部门都说“履约率”，但口径可能完全不同：

```text
销售：客户按时收到货算履约
仓储：按时出库算履约
物流：按时签收算履约
财务：按合同结算口径算履约
```

如果没有本体，Agent 可能生成“语法正确但业务错误”的查询和结论。

## 本阶段交付物

你需要产出：

1. 一个术语表，记录核心业务词。
2. 10 个 competency questions（能力问题）。
3. 一份“为什么需要本体”的问题清单。

能力问题示例：

```text
哪些 SKU 有缺货风险？
哪些订单可能延迟？
哪些供应商影响了高优先级订单？
哪些仓库服务某个区域？
哪些冷链 SKU 被分配给了非冷链承运商？
```

## 学习检查

1. 本体和数据字典有什么区别？
2. 本体和数据库 schema 有什么区别？
3. 为什么 LLM/Agent 尤其需要本体来统一业务语义？
