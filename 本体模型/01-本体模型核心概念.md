# 本体模型核心概念

## 一句话理解

本体模型是某个领域的**概念、关系、属性、约束、规则和语义口径**的正式说明书，用来让人、系统、数据库、知识图谱和 AI Agent 对同一批业务词保持一致理解。

例如供应链里：

```text
供应商 供应 物料
订单 包含 SKU
SKU 存放在 仓库
仓库 服务 区域
运输任务 交付 订单
```

本体不是普通表结构，也不是简单词典。它解决的是“业务语义如何被机器明确理解和复用”的问题。

## 四层心智模型

| 层次 | 作用 | 例子 |
|---|---|---|
| 术语表 / Glossary | 统一业务词汇 | 什么叫缺货风险、履约率、在途库存 |
| 本体 / Ontology | 定义概念、关系、属性、约束 | Supplier、SKU、Order、supplies、storedIn |
| 知识图谱 / Knowledge Graph | 存具体事实数据 | 供应商A supplies SKU123 |
| 规则与应用 / Rules & Apps | 查询、校验、推理、Agent 调用 | 计算缺货风险、解释异常、生成行动建议 |

可以记成：

```text
本体 = schema / 语义骨架
知识图谱 = data / 事实网络
规则 = policy / 业务判断
Agent = consumer / 语义使用者
```

## 核心术语

| 概念 | 含义 | 例子 |
|---|---|---|
| Class / 类 | 一类东西 | Supplier, Order, SKU |
| Individual / 实例 | 具体对象 | 供应商A, 订单1001 |
| ObjectProperty | 对象之间的关系 | 订单包含SKU |
| DataProperty | 对象自己的数据字段 | 库存数量, 订单金额 |
| Constraint | 必须满足的规则 | 冷链商品必须走冷链承运商 |
| Inference | 根据事实推出新事实 | 库存低于安全库存 => 缺货风险 |

## 本体与相近概念

| 对象 | 主要解决 | 局限 |
|---|---|---|
| 数据库表 | 存储结构化数据 | 字段含义和业务关系常常隐含在代码里 |
| 数据字典 | 解释字段含义 | 通常缺少可机器推理的关系和约束 |
| ER 模型 | 设计数据库实体关系 | 偏存储设计，不一定表达领域语义 |
| 知识图谱 | 表达实体和关系 | 需要本体作为 schema 和语义约束 |
| 本体模型 | 明确定义概念、关系、约束、推理语义 | 建模成本高，需要治理 |
