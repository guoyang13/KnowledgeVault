# 本体模型学习线路

> 目标：学习计算机和 AI 里的本体 / 本体模型，而不是哲学本体论。最终能力不是“会背 RDF/OWL 名词”，而是能把一个业务领域建成可查询、可校验、可推理、可被 LLM/Agent 使用的语义模型。

## 学习主线

```text
问题意识
→ 领域建模
→ RDF 图模型
→ RDFS / OWL
→ SPARQL
→ SHACL / 规则
→ 工程化治理
→ LLM / Agent 结合
```

## 阶段总览

| 阶段 | 目标 | 产出 |
|---|---|---|
| 0. 问题意识 | 知道本体解决什么问题 | 能区分数据表、数据字典、知识图谱、本体 |
| 1. 领域建模 | 从业务中抽象概念和关系 | 术语表、概念图、能力问题 |
| 2. RDF 图模型 | 用三元组表达事实 | Turtle 示例、实例图谱 |
| 3. RDFS / OWL | 定义类、属性、层级和可推理语义 | 可推理的领域本体 |
| 4. SPARQL | 查询 RDF 图 | 10 个常用查询 |
| 5. SHACL / 规则 | 校验数据质量和业务约束 | shape 文件、校验报告 |
| 6. 工程化 | 版本、命名、治理、工具链 | 可维护的本体项目结构 |
| 7. LLM / Agent 结合 | 让 Agent 按业务语义工作 | 本体驱动的查询、解释、校验流程 |

## 建议实践路线

先读 [[本体模型/01-本体模型核心概念]] 和 [[本体模型/02-阶段0-问题意识]]，建立“为什么需要本体”的判断力。然后边学边做 [[本体模型/12-供应链本体Mini项目]]，不要只读标准文档。每学完一个阶段，都用供应链 mini 项目产出一个小交付物。

最小目标：

```text
1. 能画出供应链领域概念图
2. 能写 30 条 RDF/Turtle 三元组
3. 能定义最小 OWL 类和属性
4. 能写 10 个 SPARQL 查询
5. 能写 5 个 SHACL 约束
6. 能让 LLM/Agent 按本体口径回答一个业务问题
```
