在 Git 中，“提交变更”是一个两步走的过程：先**暂存（Add）**，后**提交（Commit）**，最后**推送（Push）**。

为了确保万无一失，建议你养成以下标准的四步工作流：

### 第一步：查看状态 (Check Status)
在提交之前，先看看哪些文件被修改了：
```bash
git status
```
*   **红色文件**：表示已修改（Modified）或新创建（Untracked），但还没进入提交清单。

---

### 第二步：添加文件到暂存区 (Add)
你需要告诉 Git，哪些变动是你想在这次提交中包含的。
*   **添加所有文件（最常用）**：
    ```bash
    git add .
    ```
*   **添加指定文件**：
    ```bash
    git add filename.txt
    ```
*   此时再次运行 `git status`，你会发现文件名变成了**绿色**。

---

### 第三步：提交到本地仓库 (Commit)
这一步会将暂存区的文件永久保存为一个“版本记录”。
```bash
git commit -m "这里写你的提交说明"
```
*   **注意**：`-m` 后面是本次修改的描述，建议写得清晰一点，比如 `"修复了登录页面的样式"` 或 `"添加了新的算法脚本"`。

---

### 第四步：推送到远程仓库 (Push)
最后，将本地的提交记录上传到 GitHub/GitLab。
```bash
git push
```
*   如果你是第一次推送这个分支，可能需要执行：`git push -u origin main`。

---

### 💡 进阶技巧

#### 1. 合并第二步和第三步（快捷方式）
如果你修改的是**已经存在**的文件（不是新创建的文件），你可以跳过 `add` 命令，直接一步到位：
```bash
git commit -am "修改说明"
```
*(注意：这个 `-am` 不会对从未追踪过的新文件生效。)*

#### 2. 修改刚刚那次提交（后悔药）
如果你刚 commit 完发现漏掉了一个文件，或者备注写错了，可以使用：
```bash
git commit --amend -m "新的修正后的说明"
```

#### 3. 查看提交历史
确认你的提交是否成功，并查看之前的记录：
```bash
git log --oneline
```

### 🛠️ 配合好用的工具（既然你装了 iTerm2 和 Warp）

1.  **使用 LazyGit (最推荐)**：
    在你之前安装的 `lazygit` 中，你只需要：
    *   按 `j/k` 选中文件，按 `空格` (Add)。
    *   按 `c` (Commit)，输入描述并回车。
    *   按 `P` (Push)。
    *   *这比背命令直观得多。*

2.  **使用 Warp AI**：
    如果你不确定该怎么描述你的改动，可以在 Warp 输入框输入：
    `# generate a commit message for my changes`
    Warp 会分析你的代码改动并帮你写好 `commit -m` 的内容。

**总结口诀：**
`git add .` —— 进站（装货）
`git commit -m "..."` —— 封箱（打包）
`git push` —— 发货（送走）

**你现在的变动大吗？如果不确定有没有大文件，建议先执行 `git status` 确认一下。**
